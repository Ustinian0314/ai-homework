# BytePlus RDS PostgreSQL 設置完整指南

## 第一步：註冊BytePlus賬號

1. **訪問BytePlus官網**
   - 前往：https://www.byteplus.com/
   - 點擊右上角"註冊"或"Sign Up"

2. **完成註冊**
   - 填寫郵箱、密碼等基本信息
   - 驗證郵箱地址
   - 完成身份驗證

## 第二步：開通RDS服務

1. **登入BytePlus控制台**
   - 前往：https://console.byteplus.com/
   - 使用你的賬號登入

2. **找到RDS服務**
   - 在控制台首頁搜索"RDS"或"數據庫"
   - 或者在產品列表中選擇"Relational Database Service"

3. **開通RDS服務**
   - 點擊"立即開通"或"Create Instance"
   - 選擇PostgreSQL數據庫引擎
   - 選擇合適的配置（建議先選擇最低配置用於測試）

## 第三步：創建PostgreSQL實例

1. **選擇地域**
   - 選擇離你最近的地域（如香港、新加坡等）

2. **配置實例**
   - **實例名稱**：輸入一個有意義的名稱，如"ai-workspace-db"
   - **數據庫引擎**：選擇PostgreSQL
   - **版本**：選擇最新穩定版本（如PostgreSQL 14或15）
   - **規格**：選擇最低配置即可（如1核2GB）

3. **網絡配置**
   - **VPC**：選擇默認VPC或創建新的
   - **子網**：選擇合適的子網
   - **安全組**：確保開放PostgreSQL端口（5432）

4. **數據庫配置**
   - **數據庫名稱**：輸入數據庫名稱，如"aiworkspace"
   - **用戶名**：設置數據庫管理員用戶名，如"admin"
   - **密碼**：設置強密碼（記住這個密碼！）

5. **完成創建**
   - 檢查所有配置
   - 點擊"創建實例"或"Create"
   - 等待實例創建完成（通常需要5-10分鐘）

## 第四步：獲取連接信息

1. **進入實例詳情**
   - 在RDS控制台找到你創建的實例
   - 點擊實例名稱進入詳情頁面

2. **查看連接信息**
   - **主機地址**：在"連接信息"或"Connection Info"中查看
   - **端口**：通常是5432
   - **數據庫名稱**：你之前設置的數據庫名
   - **用戶名**：你設置的管理員用戶名
   - **密碼**：你設置的密碼

3. **測試連接**
   - 在控制台中找到"連接測試"功能
   - 或者使用數據庫管理工具（如pgAdmin）測試連接

## 第五步：配置安全組

1. **找到安全組**
   - 在實例詳情頁面找到"安全組"或"Security Group"

2. **添加入站規則**
   - 點擊"編輯規則"或"Edit Rules"
   - 添加入站規則：
     - **類型**：PostgreSQL
     - **端口**：5432
     - **源**：0.0.0.0/0（允許所有IP，生產環境建議限制特定IP）

## 第六步：更新代碼配置

編輯 `user-api.js` 文件，將以下信息替換為你的實際信息：

```javascript
const pool = new Pool({
  user: 'admin',                    // 替換為你的數據庫用戶名
  host: 'your-instance-host',       // 替換為你的實例主機地址
  database: 'aiworkspace',          // 替換為你的數據庫名稱
  password: 'your-password',        // 替換為你的數據庫密碼
  port: 5432,                       // 通常是5432
  ssl: {
    rejectUnauthorized: false       // 雲端數據庫通常需要SSL
  }
});
```

## 第七步：測試連接

1. **啟動API服務**
   ```bash
   npm install
   npm start
   ```

2. **檢查控制台輸出**
   - 如果看到"已連接到PostgreSQL數據庫"，說明連接成功
   - 如果看到錯誤信息，請檢查配置

3. **測試API**
   - 訪問：http://localhost:3001/api/health
   - 應該返回健康檢查信息

## 常見問題解決

### 1. 連接被拒絕
- 檢查安全組是否開放5432端口
- 確認實例狀態是否為"運行中"
- 檢查用戶名和密碼是否正確

### 2. SSL錯誤
- 確保 `ssl: { rejectUnauthorized: false }` 配置正確
- 某些雲端數據庫需要SSL連接

### 3. 找不到實例
- 確認選擇了正確的地域
- 檢查實例是否創建成功
- 等待實例完全啟動

## 費用說明

- BytePlus RDS通常有免費試用額度
- 最低配置實例費用相對較低
- 建議先使用最低配置進行測試

## 安全建議

1. **密碼安全**：使用強密碼
2. **IP限制**：生產環境建議限制特定IP訪問
3. **定期備份**：啟用自動備份功能
4. **監控**：啟用數據庫監控和告警
